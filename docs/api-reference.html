Complete reference documentation for the CUSS2.ts TypeScript SDK

Cuss2 Class

Main SDK class for CUSS2 platform interaction. Manages connection, state transitions, and component orchestration.

Static Methods

connect()

static 
Cuss2.connect(
    clientId: string,
    clientSecret: string,
    wss?: string,
    deviceId?: string,
    tokenUrl?: string
): Cuss2

Creates and connects to a CUSS2 platform instance.

  • clientId string
    OAuth client identifier
  • clientSecret string
    OAuth client secret
  • wss string
    WebSocket URL (default: "ws://localhost:22222")
  • deviceId string
    Device identifier (default: "00000000-0000-0000-0000-000000000000")
  • tokenUrl string
    OAuth token endpoint (defaults to wss + "/oauth/token")

Example

const cuss2 = Cuss2.connect(
    "client-id",
    "client-secret",
    "wss://platform.example.com",
    "device-123",
    "https://oauth.example.com/token"
);

await cuss2.connected;

Application States

CUSS2 application lifecycle states with transition rules.

enum ApplicationStateCodes {
    STOPPED = "STOPPED",
    INITIALIZE = "INITIALIZE", 
    UNAVAILABLE = "UNAVAILABLE",
    AVAILABLE = "AVAILABLE",
    ACTIVE = "ACTIVE"
}

State Transition Rules

From State Valid Transitions
STOPPED → INITIALIZE
INITIALIZE → UNAVAILABLE
UNAVAILABLE → AVAILABLE, INITIALIZE
AVAILABLE → ACTIVE, UNAVAILABLE
ACTIVE → AVAILABLE, UNAVAILABLE
Any State → STOPPED

Error Codes

Standard CUSS2 platform error codes for error handling.

Code Name Description
1000 DEVICE_NOTOPERATIONAL Device is not operational
1001 MEDIA_NOTREADY Media not ready (e.g., out of paper)
1002 DEVICE_BUSY Device is busy processing
1003 INVALID_DATA Invalid data format provided
1004 TIMEOUT Operation timed out
1005 PERMISSION_DENIED Permission denied for operation
1006 INVALID_STATE Invalid state for requested operation

Error Handling Example

try {
    await cuss2.boardingPassPrinter.enable();
} catch (error) {
    // Check error code directly
    if (error.code === 1001) {  // MEDIA_NOTREADY
        console.log("Printer out of paper");
        // Handle paper reload
    } else if (error.code === 1000) {  // DEVICE_NOTOPERATIONAL
        console.log("Printer not operational");
        // Handle printer offline
    } else {
        console.error("Unexpected error:", error);
    }
}
